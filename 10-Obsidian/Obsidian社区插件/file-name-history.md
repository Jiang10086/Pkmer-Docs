## 概述

### 插件名称
File Name History

### 主要功能
跟踪Obsidian中Markdown文件的文件名和父文件夹名变更，将旧名称存储在配置的列表属性（如别名）中。

### 适用场景
适用于生成重定向规则，与Astro Composer插件及Astro Modular博客主题配合使用效果更佳，也是Vault CMS项目的一部分。

### 核心特色
1. 支持文件名和父文件夹名变更跟踪，可配置超时时间。
2. 可设置忽略模式，跳过匹配正则模式的名称。
3. 能切换大小写敏感性，支持文件夹过滤。
4. 默认跟踪 `.md` 文件，也支持自定义扩展名。
5. 可配置存储历史的属性，能自动创建缺失属性。

### 使用建议
通过Obsidian的社区插件搜索安装该插件，路径为：设置>社区插件（若未启用需先启用），搜索“File Name History”并点击安装。
