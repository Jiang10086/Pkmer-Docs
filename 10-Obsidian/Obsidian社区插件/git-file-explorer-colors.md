## 概述

### 1. 主要功能
为 Obsidian 文件资源管理器依据 Git 状态进行颜色标记，如新增或未跟踪文件显示为绿色，修改文件为柔和橙色，删除路径的文件夹信号为柔和红色，还能将子项变化反映到父文件夹，可在多种事件下刷新，也有手动刷新命令。

### 2. 适用场景
适用于使用 Obsidian 作为 AI 第二大脑、且将保险库存于单个 Git 仓库的桌面用户，能帮助快速发现 Markdown 文件的变化。

### 3. 核心特色
无需将 Obsidian 转变为完整 IDE 就能直观看到文件变化，仅读取 Git 状态进行颜色标记，不改变 Git 状态。

### 4. 使用建议
需使用 Obsidian 桌面版，系统路径中要安装 `git`，保险库需是 Git 仓库或位于单个 Git 仓库内，且通过桌面应用进行本地文件系统访问。可通过社区插件或手动方式安装。
