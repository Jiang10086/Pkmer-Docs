# Obsidian 插件：Properties Filename

> [!Note] 插件名片
> - 插件名称：Properties Filename
> - 插件作者：gilles6
> - 插件说明：这个插件还没经过Obsidian官方人员的人工审核。它能在笔记的前置元数据属性（比如姓氏 + 名字）发生变化时，自动对笔记进行重命名。

## 概述

### Properties Filename插件总结
1. **主要功能**：依据笔记的前置元数据属性自动重命名笔记文件，当笔记属性变化时，按配置模板重命名文件，同时Obsidian的重命名API会自动更新指向该文件的维基链接。
2. **适用场景**：适用于维护联系人、患者或客户笔记等场景，这些笔记的规范标识符由属性组合而成，如姓名、项目代码加标题等，编辑属性即可自动更新文件名，无需手动重命名。
3. **核心特色**：可配置重命名规则，包括应用的文件夹、文件名模板和可选的类型要求；提供手动重命名命令，支持批量重命名；自动重命名默认关闭，可在设置中开启。
4. **使用建议**：若需自动重命名，可在设置中开启自动重命名功能；若想手动触发，可使用命令“Properties Filename: Rename current file from properties”重命名当前文件，或用“Properties Filename: Rename all matching files in vault”批量重命名匹配规则的文件。安装可手动下载文件放入指定目录，也可通过BRAT插件安装。
